# 《商品链接增长诊断方法论 V1.0》

这套方法论的核心不是“商品链接诊断”，而是：

> 用 GMV 拆解确定增长问题，用生命周期判断商品阶段，用人群与流量匹配定位根因，用转化与利润验证商品价值，最后把诊断结论转成增长任务。

换句话说：

> 商品链接诊断 = GMV增长归因 × 商品生命周期 × 人群流量匹配 × 转化利润验证 × 任务闭环。

---

# 一、总述

1. 商品诊断不是看数据，而是“问题 -> 方案 -> 任务 -> 增长”，并且要按生命周期、诊断维度、指标对标、问题排查、方案执行、数据验证形成闭环。
2. 链接诊断拆成新品期、成长期、成熟期、爆款期、衰退期，并给出各阶段的核心目标、关键指标和诊断动作。

---

# 二、方法论总定义

## 商品链接增长诊断方法论

商品链接增长诊断，是一套以 GMV 结果为起点，以商品生命周期为主线，以人群匹配为核心，以流量、点击、转化、利润、竞品、体验为诊断维度，最终生成增长任务并验证结果的经营判断系统。

它解决五个连续问题：

```text
1. GMV为什么变好或变差？
2. 当前商品处在哪个生命周期？
3. 这个阶段最核心的问题是什么？
4. 问题出在流量、人群、点击、转化、利润、竞品还是体验？
5. 今天应该先做什么，做完看什么数据证明有效？
```

---

# 三、方法论的底层公式

GMV = 访客数 × 转化率 × 客单价。

| GMV杠杆 | 代表问题 | 经营语言 |
| --- | --- | --- |
| 访客数 | 有没有人来 | 流量问题 |
| 转化率 | 来了买不买 | 承接问题 |
| 客单价 | 买得多不多、贵不贵 | 价值放大问题 |

所以商品链接诊断的第一步，不是直接看主图、详情页、投放，而是先判断 GMV 变化到底来自访客数、转化率还是客单价。

---

# 四、GMV增长归因层

## 1. 访客数问题：有没有足够多的人进来

访客数可以继续拆为：访客数 = 展现量 × 点击率，也可以按来源拆为：免费流量 + 付费流量 + 自主访问流量。

访客数下降时，不能直接说“流量不够”，而要继续判断：

| 问题类型 | 具体含义 |
| --- | --- |
| 展现不足 | 平台没有把商品展示给足够多人 |
| 点击率低 | 展示了，但用户不愿意点 |
| 搜索下降 | 人找货入口失效 |
| 推荐下降 | 货找人入口失效 |
| 付费下降 | 推广计划失效或预算不足 |
| 自主访问下降 | 老客、收藏、私域、品牌心智下降 |

因此，访客数诊断的本质是：判断商品是“没被看见”，还是“被看见但没人点”。

## 2. 转化率问题：用户进来了为什么不买

转化率不能粗暴看一个总数，要拆成：下单转化率 × 支付转化率。同时结合商品链接链路：点击 -> 加购 -> 下单 -> 支付 -> 询单 -> 售后。

| 类型 | 问题本质 | 重点排查 |
| --- | --- | --- |
| 下单转化问题 | 用户还没有被说服 | 主图、详情页、SKU、价格、评价、买家秀、问大家、客服 |
| 支付转化问题 | 用户已经想买，但最后没付 | 优惠门槛、库存、价格变化、催付、活动规则、支付体验 |

转化诊断的本质是：判断用户是“没被打动”，还是“临门一脚流失”。

## 3. 客单价问题：用户买得少还是买得便宜

客单价可以拆成：客单价 = 件单价 × 连带购买件数。

| 路径 | 经营动作 |
| --- | --- |
| 提升件单价 | 升级款、高端款、价值包装、价格带上移 |
| 提升连带购买件数 | 搭配购、满减、套装、顺手买一件、客服推荐 |

客单价诊断的本质是：判断商品有没有把单个用户价值吃透。

---

# 五、商品生命周期判断层

生命周期不只是时间划分，而是经营任务划分。

| 阶段 | 核心任务 | 方法论关键词 |
| --- | --- | --- |
| 新品期 | 验证定位，打准标签，跑通基础转化 | 定标签 |
| 成长期 | 扩大流量，提升转化，建立承接能力 | 抓增长 |
| 成熟期 | 稳定自然流量、利润结构和复购 | 稳效率 |
| 爆款期 | 保权重、防竞品、建壁垒 | 建护城河 |
| 平销期 | 判断利润、动销、节点价值 | 判价值 |
| 衰退期 | 判断盘活、清仓或淘汰 | 定生死 |

不同阶段有不同主要矛盾。新品期优先看标签、基础点击和初始转化；成长期优先看流量放大、转化承接和增长效率；成熟期优先看自然流量稳定性、利润结构和复购。

---

# 六、商品角色判断层

同样是成长期商品，有的要打爆，有的要赚钱，有的要引流，有的只负责动销。

| 商品角色 | 核心价值 | 诊断重点 |
| --- | --- | --- |
| 流量款 | 带来访客和新客 | 点击率、搜索/推荐流量、拉新效率 |
| 爆款 | 打排名、打权重、带动店铺势能 | 转化率、类目排名、免费流量、竞品防御 |
| 利润款 | 贡献利润 | 毛利率、客单价、ROI、高价值人群 |
| 动销款 | 稳定店铺基础成交 | 稳定成交、低维护成本、库存健康 |
| 活动款 | 承接活动和节点流量 | 价格力、库存、活动转化、风险控制 |

如果商品角色判断错了，后面的指标判断都会错。

---

# 七、诊断闭环

商品链接诊断必须形成“问题-指标-标准-动作-验证”的闭环：

1. 先判断问题类型：流量不足、点击率低、转化承接弱、客单价弱、利润结构差、人群不匹配。
2. 再绑定判断依据：GMV、访客数、CTR、CVR、客单价、毛利率、ROI、停留时长、加购率、支付转化率。
3. 再明确判断标准：低于行业均值、连续 N 天下滑、CVR 低于行业均值 80%、CTR 高但 CVR 低等。
4. 再生成解决动作：优化标题关键词、调整主图卖点、重构详情页、优化 SKU、调整推广人群、补强评价与买家秀。
5. 最后验证效果：观察 CVR、CTR、GMV、ROI、加购率、支付转化率是否提升，并判断是否达到行业均值或阶段目标。

---

# 八、触发条件与终止条件

触发条件示例：

- 日访客达到可观察样本量，且 CVR 低于行业均值的 80%，持续 5 天以上。
- CTR 正常但转化率下滑，说明承接能力可能不足。
- GMV 下滑时，先拆访客数、转化率、客单价，再进入对应诊断分支。

终止条件示例：

- CVR 回升到行业均值以上，且连续观察周期稳定。
- 优化后 GMV、ROI 或核心阶段指标达到目标。
- 多轮优化收益低于阈值，转入维持或重新判断商品角色。

---

# 九、可复用模板

商品链接诊断表应至少包含：

- 商品基础信息、生命周期阶段、商品角色。
- 当前 GMV、访客数、CTR、CVR、客单价、ROI。
- 问题类型、判断依据、判断标准。
- 解决动作、负责人、执行时间。
- 验证指标、观察周期、结果结论。
